系统结构

B / S架构：（以后主要走的方向是这个）

​ Browser / Server （浏览器 / 服务器）

优点：升级方便，只升级服务器端代码即可

缺点：速度慢，体验不好，界面不炫酷

C / S架构

​ Client / Server （客户端 / 服务器端）

优点：速度好，体验好，界面炫酷（娱乐型的系统多数是C/S架构）

缺点：升级麻烦，维护成本较高
